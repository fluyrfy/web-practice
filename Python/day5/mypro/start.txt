==================================
webpack對圖片的打包機制
webpack有一個好處:會在打包時先閱讀所有的代碼， 把用到的圖片打包起來， 放在dist目錄中

npm run build

打包後的代碼會放在dist目錄下
導致問題: 如果圖片的地址是通過網路動態加載生成的， 則webpack打包的時候不會打包到項目中， 導致圖片無效

vue的目錄有一個Public的文件夾 放在這個文件夾下的內容 忽略webpack檢查機制 都會被打包

將src/assets下方的圖片文件夾拷貝到public下方
